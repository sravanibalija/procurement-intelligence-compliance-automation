# Procurement Intelligence & Compliance Automation System

## Overview
This project simulates an end-to-end procurement analytics solution designed to improve spend visibility, monitor supplier performance, detect compliance exceptions, and support data-driven procurement decision-making.

## Objective
To build a scalable analytics workflow that combines procurement spend analysis, supplier performance tracking, compliance monitoring, and reporting automation using SQL, Python, and Tableau/Power BI.

## Tools Used
- SQL
- Python
- pandas
- Excel / CSV
- Tableau / Power BI

## Key Features
- Procurement spend analysis by supplier, category, department, and month
- Supplier scorecard with on-time delivery, defect rate, and lead time metrics
- Compliance exception monitoring using business-rule-based flags
- Automated KPI generation and report-ready output files
- Dashboard views for executive, operational, and compliance reporting

## Business Questions Addressed
- Which suppliers account for the highest spend?
- Which procurement categories drive the greatest cost?
- Which suppliers are underperforming on delivery or quality?
- Where are compliance gaps occurring in procurement transactions?
- What reporting areas can be automated to reduce manual effort?
